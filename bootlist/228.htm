2) Problem Installing MS XP Updates - Continued

Eugene G. emailed:
Subject: BootLIST 227 Item 3) Problem Installing MS Updates

Recently I have encountered several PCs which do not update (all updates are shown as 0 KB 0 minutes) - I don't know what causes this problem. For most, I found running the following batch file did the trick:

echo on
net.exe stop wuauserv
cd /d %windir%
rd /s softwaredistribution
REGSVR32 %windir%\system32\qmgr.dll
REGSVR32 %windir%\system32\qmgrprxy.dll
REGSVR32 %windir%\system32\wuapi.dll
REGSVR32 %windir%\system32\wuaueng1.dll
REGSVR32 %windir%\system32\wuaueng.dll
REGSVR32 %windir%\system32\wucltui.dll
REGSVR32 %windir%\system32\wups.dll
REGSVR32 %windir%\system32\msxml3.dll
net.exe start wuauserv
net.exe stop cryptsvc
rename %windir%\system32\catroot oldcatroot
rename %windir%\system32\catroot2 oldcatroot2
net.exe start cryptsvc


*** Item 3, Issue 227 was very basic, but it's easy sometimes to overlook the basics. In addition to your batch file I also have this to refer to:

http://www.bootdisk.com/xptop20.htm#4

3) Cant Replace Corrupt Win98 Files While In Use

michael.a. posted - When my computer starts windows it comes up with a message box saying that the following system files are corrupt :

DDEML.DLL
MCICDA.DRV
WINASPI.DLL

Windows still is running fine and I have replacement files for them but I cant replace them because they are in use. I tried using SFC and it doesn't work. I have windows 98 SE.


*** Try replacing them after booting to Safe Mode, or, after booting with a basic bootdisk. If you use a bootdisk windows has no chance of running so they cant be in use. Still, even if that works, you may have to re-register them as two of them are windows dlls.

http://word.mvps.org/FAQs/Customization/RegisterDLL.htm

Regarding SFC - System File Checker in win98 I too haven't found that utility to be of much use.

Jim B. said:

Sometimes the tools below will find files which they are unable to delete because they are in use.

A program called Copylock, here:

http://noeld.com/programs.asp?cat=misc

Can aid in the process of replacing, moving, renaming or deleting one or many files which are currently in use (e.g. system files like comctl32.dll, or virus/trojan files.)

Another is Killbox, here:

http://www.downloads.subratam.org/KillBox.zip

A third which is a bit different but often very useful is Delete Invalid File, here:

http://www.purgeie.com/delinv.htm

*** The above utils should work in both win98 and XP.

4) USB Or Firewire For External Drives

Advid wrote - I want to add an external hard disk and external DVD writer to my XP PC. If these have a choice of either USB2 or Firewire - does one outweigh the other? Is firewire connection the same as USB (Win XP home) - ie Do I just plug in and Win will see the 'extra' drives. Or do I need any extra software/CD to install it

Oxford Systems answered:
Look for some to disagree on this but current generation Firewire would be the preferable to USB2. While the theoretical transfer rate of USB2 is faster (480mpbs vs. 400 mbps for Firewire), in practice Firewire is a more efficient protocol with faster access and transfer rates in the real world.

If your system has Firewire ports, you can just plug it in. Firewire support is built into Win XP. Be aware that Win XP SP2 upgrades "broke" Firewire and caused some very slow transfer rates. There is an explanation and a fix for it here:

http://support.microsoft.com/?kbid=885222

Edward W. T. adds:
Re USB2 or Firewire, it depends. If you intend to connect the external drive to a variety of machines then USB2 is more 'universal' with respect to ports. Firewire ports can vary between machines to none to 4 pin or 6 pin. Firewire with a 4 pin port will require a separate power supply for the HDD

With regards to software, if you are using WINXP SP2 and a motherboard with USB2 and firewire ports built in the WINXP native drivers will be sufficient. If you are connecting through an add in card then the card supplier may require special drivers to be installed.


*** External drives are used for additional storage and data backup, and are designed for portability and use on other systems when you need to get data back. To that end I'd suggest going with a USB as most all PCs these days have USB but only a small percentage have Firewire.

5) Recommended First Step After Installing XP

Will proclaimed - I just installed Windows XP version 2 from the new Microsoft CD on a new hard drive. The geeks at work said the first thing I should do is "tweak it up" to make it run better. I dont have a problem with the way it runs now and it boots up nice and fast compared to my old XP.


*** I'm not sure what they actually mean by "tweak it up", tho I have my suspicions. The VERY first thing to do, assuming you are not behind a hardware firewall like your inhome router, is to make sure you have a firewall running, at least for inbound traffic. So, if you have a firewall program on CD or saved to CD, install it. If not, then IMMEDIATELY enable or make sure the built in XP firewall is running.

http://support.microsoft.com/?kbid=283673

And I'd certainly would not mess with system aka "tweak it up" if I was already happy with how it was running.
