2) Delayed Write Failed

Bendo writes - Windows XP-Delayed Write Failed. Windows was unable to save all the data for the file [filename]. The data has been lost. This error may be caused by a failure of your computer hardware or network connection. Please try to save this file elsewhere. How do I solve this problem?

PcEngWork answers:
Most modern drives employ Write behind caching. This means that data to be written to the drive is held temporarily, until the drive is available for writes and then the data is flushed to the drive.

Whenever this happens either the disk or the controller failed to write the data to the drive. You should look at the System Event log, as these types of errors get logged there. It is also a good idea to see how many instances of this error occurred. Sometimes it's a single event that may not happen again for a long time - or it could be an indication of a drive beginning to fail.

I would recommend you do a Chkdsk C: /R to verify the integrity of the drive. IDE controllers fail, but not that often so it's most likely the drive that triggered the fault. Just do the check disk and monitor the System logs to see if additional errors of these type happen in the future. A reboot is in order as well, to save your Registry/Settings to the drive.

3) My Password Doesn't Work For Administrator Logon

NoWeaKnESS inquires - I am attempting to repair my system. So I pop in the WinXP CD, select the repair option etc etc... then it asks me to enter an administrator's password. My password doesn't work. Is there a set admin password? Or what is going on?

Bruce C. answers:
The password being requested is that of the built-in Administrator account, not the password for any other administrative user accounts you may have created. Unless you've deliberately set the Administrator's password via the Management Console (_not_ the Control Panel), it's probably still blank. WinXP Pro asks you to set the Administrator's password during installation, but WinXP Home doesn't.


*** In other words just hitting Enter should work for blank passwords :)

5) I Swear I Don't Have Two CPUs

Tim questions - Ok, I recently changed my CPU in my PC. I was able to boot up my computer fine and my new processor was detected fine by both my motherboard and WinXP, except, WinXP thinks that I have two of them. My motherboard doesn't even have two slots. I am not used to having two CPU monitoring graphs in the task manager My specs are as follows:

Processor: P4 2.8GHz, 800MHz FSB, HT Tech, Prescott make
Motherboard: ASRock P4S55FX+

R. M. responds:
Your processor is a Hyperthreaded CPU. It's one physical processor, functioning in XP as two independent CPUs. This is normal, you don't need to make any changes. However, you can turn off Hyperthreading support in the Motherboard BIOS settings. But if you do, you'll loose HT support for the programs that support it.

Miha P. adds:
You really have only one processor, but it is built on Hyper Threading technology (that is what that HT Tech means in your specification). That is why you see two processors in Task Manager. What you can do in Task Manager under Performance Tab click on View | CPU History | One Graph for all CPUs.
